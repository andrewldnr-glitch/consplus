# PRD — Learn (MVP)

**Продукт:** Learn — премиальное мобильное (web) приложение для обучения и тестов по пакетным предложениям КонсультантПлюс.  
**Платформа:** Static Web (GitHub Pages) + Telegram WebView (mobile-first).  
**Данные:** локальная база знаний (JS-файл) + локальные тесты/генерация вопросов.  
**Источник базы:** «ИНФОРМАЦИОННОЕ СОДЕРЖАНИЕ (Смарты_ИЮНЬ_2025г.)».

---

## 1) Проблема

Текущая версия “табличная”, навигация находится наверху, в Telegram приходится постоянно скроллить вверх/вниз, фон выглядит “рваным”.  
Также тесты могут быть **логически некорректными**, если пункт входит в несколько пакетов: пользователь отвечает “верно по смыслу”, а тест считает “неверно”.

---

## 2) Цели MVP

1. **Премиальный UI** в стиле “FinancePro”: dark + purple, карточный интерфейс, минимализм, аккуратные тени и glow.
2. **Удобная навигация**: bottom navigation + безопасные отступы (safe-area) для iOS.
3. **Knowledge Base**: пакеты → уровни → “что входит” (список банков), поиск по ключевым словам.
4. **Тесты без логических ошибок**:
   - если правильных вариантов несколько → вопрос становится **multiple choice**;
   - отдельный режим “отличия уровней” (Базовый → Оптимальный), где ответ однозначен.
5. **Результаты**: корректность ответа показывается сразу после выбора/подтверждения; в конце — % верных/неверных.
6. **Статистика**: история попыток + график точности (7 дней) + streak.

---

## 3) Не-цели (пока)

- Настоящая авторизация, синхронизация, серверная аналитика.
- Оплаты/подписки (paywall — только UX-экран, без реального платежа).
- Контент-редактор в приложении (редактирование делается в `data/content.js`).

---

## 6) Ключевые сценарии

- Открыть приложение → Home → “Фокус” (Базовый/Оптимальный) → открыть пакет → посмотреть состав.
- Поиск по банкам (например “ФАС”) → увидеть, где встречается.
- Пройти тест:
  - “Специализация пакетов” (банк → пакет),
  - “Апгрейд Базовый→Оптимальный” (что добавляется),
  - “Экзамен 30 вопросов”.
- Посмотреть статистику и слабые места.

---

## 7) Метрики успеха (MVP)

- Time-to-answer: время до нахождения состава пакета (снижение).
- Test accuracy: рост точности со временем.
- Retention: 7-day streak/повторные сессии.
- Доля завершенных тестов.

---

## 8) Assumptions

- Пользователь готов обновлять контент через файл (`data/content.js`) и деплой на GitHub Pages.
- Telegram WebView корректно рендерит CSS blur/градиенты (с graceful degradation).

---

## 9) Риски и как закрываем

- **Сложные/тяжёлые эффекты в CSS** → используем мягкие тени/градиенты и отключаем “тяжёлые” эффекты при необходимости.
- **Логическая неоднозначность тестов** → правило: multiple-choice, либо вопросы “на отличия уровней”.
- **Расхождение контента с реальностью** → держать один источник правды (docx → обновление `data/content.js`).

---

## 10) Release plan

- MVP: Knowledge Base + Tests + Results + Stats + Premium screen (демо) + Docs.
- v1: подбор “слабых тем”, рекомендации, больше типов вопросов в прод.
- v2: аккаунт, облачная синхронизация, полноценная подписка.
