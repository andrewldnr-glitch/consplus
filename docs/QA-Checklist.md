# QA Checklist — Learn (MVP)

Цель: быстро проверить ядро сценариев (smoke) и стабильность (регрессия) на мобильных.

---

## 1) Environments

- iOS Safari / Telegram WebView (основной риск)
- Android Chrome / Telegram WebView
- Desktop Chrome (для отладки)

---

## 2) Smoke tests (ядро)

### 2.1 Launch & navigation

- [ ] `#/splash` открывается, анимация не лагает
- [ ] автопереход на onboarding (если новый пользователь) / home (если onboarded)
- [ ] bottom nav переключает вкладки без “прыжков”
- [ ] safe area: нижняя навигация не перекрывает контент

### 2.2 Onboarding

- [ ] 3 шага переключаются
- [ ] выбор goal сохраняется (перезапуск → цель сохранена)
- [ ] “Готово” ведёт на Home

### 2.3 Knowledge Base

- [ ] поиск работает (Enter + кнопка)
- [ ] toggle “focus only” работает
- [ ] открытие пакета
- [ ] вкладки уровней переключаются и меняют состав
- [ ] “Скопировать ссылку” работает (если clipboard недоступен → показывается toast)

### 2.4 Tests

- [ ] открыть Tests Hub
- [ ] открыть тест → Start → тест начинается
- [ ] Single choice вопрос:
  - [ ] выбор подсвечивает correct/wrong
  - [ ] появляется explanation + “Дальше”
- [ ] Multiple choice вопрос:
  - [ ] выбор нескольких вариантов
  - [ ] “Проверить” подсвечивает
- [ ] Results:
  - [ ] корректный % и счет
  - [ ] “ещё раз” ведёт к превью теста
  - [ ] “статистика” ведёт в Stats

### 2.5 Stats

- [ ] график рисуется даже при пустой истории
- [ ] streak считается корректно
- [ ] список дней корректный

### 2.6 Profile / Premium

- [ ] смена goal
- [ ] reset сбрасывает историю и ведет на splash
- [ ] premium toggle сохраняется
- [ ] paywall открывается и кнопки работают (демо)

---

## 3) Regression core (критические проверки)

- [ ] hash routing: прямое открытие `#/package/:id?v=...` работает
- [ ] hash routing: прямое открытие `#/test/:id` работает
- [ ] localStorage: corrupt state → приложение не падает (фоллбек на defaultState)
- [ ] длинные списки “что входит” прокручиваются плавно, без “дрожания”
- [ ] поиск по длинным строкам не ломает layout

---

## 4) UI/UX quality

- [ ] текст не обрезается в topbar (ellipsis работает)
- [ ] кликабельные области ≥ 44×44
- [ ] цвета корректные в dark mode
- [ ] “glow” не превращает интерфейс в “неон‑игрушку”

---

## 5) Known limitations (MVP)

- clipboard API может быть недоступен в некоторых WebView
- markdown docs в Telegram могут открываться как plain text
